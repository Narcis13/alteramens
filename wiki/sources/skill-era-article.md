# The Skill Era of the Internet

Manifesto article about the paradigm shift from the API era to the Skill era. Source: [[workshop/drafts/Articol interesant|Vault concept note]].

## The Shift

**API Era (last 15 years):** Build a product, expose an API. Companies that won owned the pipes (Stripe=payments, Twilio=messaging, SendGrid=email). Distribution = convincing developers to integrate.

**Skill Era (now):** LLMs compress execution. What's scarce is judgment, not function. You build expertise and package it as a skill. Distribution = becoming part of agent workflows.

## Key Distinction: API vs Skill

| API | Skill |
|-----|-------|
| Doorway into a function | Doorway into judgment |
| "Here's how to send an email" | "Here's how to audit a landing page like a growth operator" |
| Precise, mechanical, bounded | Encodes a way of thinking |
| Scales by developer integrations | Scales by agent invocations |

## New Playbook

Old: Build SaaS → Design UI → Onboard users → Drive retention → Expand seats

New: **Encode a high-leverage playbook → Package as skill → Let agents call it thousands of times/day**

The interface shrinks. The leverage expands.

## Evolution

1. Software was the executor
2. Software is the orchestrator
3. Expertise becomes infrastructure

## Company Design

New companies look surprisingly small: tight repo, handful of skill files, 2-5 people. But those skills sit inside thousands of workflows, shaping decisions at scale.

**"A new class of companies built less around dashboards and more around encoded judgment."**
