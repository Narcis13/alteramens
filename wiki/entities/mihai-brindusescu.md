# Mihai Brîndușescu — User Zero

Narcis's son. Future candidate for admission to UMF Carol Davila București (cohort 2027). Designated **user zero** for the AI tutor for medical admission product.

## Role

Mihai is not the reason for the product — he is the **proof point**. The brainstorm on 2026-04-07 explicitly resolved this distinction: Narcis's identity is "founder of an ed-tech startup," not "father building a tool for his son." Product decisions optimize for product viability; Mihai's role is to be the first real user whose preparation is run end-to-end through the system.

This dual role creates the [[authentic-creation]] advantage: a founder building publicly the instrument his own child uses for a high-stakes, irreversible exam outcome. Skin in the game is structural, not rhetorical.

## Implications

- **Build-in-public stake:** Narcis confirmed "foarte mult" — willing to bet publicly that Mihai uses the tool as primary preparation method. This unlocks [[building-in-public]] as a marketing channel with maximum credibility.
- **Live test loop:** Mihai's daily usage produces real feedback on calibration accuracy, agent behavior, parent dashboard usefulness, and longitudinal model quality. No focus group can match this.
- **Privacy boundary:** content shared publicly about Mihai's progress requires his ongoing consent. This needs to be explicit and revocable. Parent dashboard data is for Narcis as parent, not for marketing.
- **Outcome commitment:** the public demo culminates on the actual admission result. A success creates a definitive case study; a failure must be handled with honesty rather than spin.
