# Leverage — Force Multipliers

Getting bigger outcomes from the same input. The mechanism that turns specific knowledge into wealth at scale.

## Four Types

| Type | Description | Permission Needed? |
|------|-------------|-------------------|
| People | Others working for you | Yes (hire, manage) |
| Capital | Money working for you | Yes (raise, borrow) |
| **Content** | Media working for you | **No (permissionless)** |
| **Code** | Software working for you | **No (permissionless)** |

**Code and content are permissionless leverage.** You don't need anyone's permission to write code or create content. This is why they're the best starting point for a solo builder.

## Permissionless Leverage in Practice

> "If you can't code, write books and blogs, record videos and podcasts." — Naval

For Alteramens:
- **Code leverage:** 10 projects, each potentially serving thousands
- **Content leverage:** Building in public, skills for AI agents
- **AI leverage (new):** Claude Code as force multiplier — projects that would take months become weeks

## The Skill Era Extension

In the [[skill-era]], leverage evolves:
- **API Era:** Leverage through developer integrations (seats)
- **Skill Era:** Leverage through agent invocations (thousands of calls/day)

The interface shrinks, the leverage expands. A handful of skill files can shape decisions across hundreds of companies.

## Decision Implication

Every project should maximize leverage: prefer code over manual services, prefer platforms over one-off deliverables, prefer skills over dashboards.
